## 二点五、V6.3 天赋树 → 职场硬衔接

> **核心变更**：V6.3改为硬锁死专业后，职业路径也受到更明确的约束。选物理方向=理工科就业面，选历史方向=文科就业面。跨方向转行需在工作阶段额外付出代价。

### 天赋终极节点 → 职业路线

| 天赋终极节点 | 可进入职业路线 | 起点加成 | 约束说明 |
|-------------|:--:|:--:|------|
| 📐「万物皆数」 | 💻技术型、⚖️平衡型 | +10% | 理工背景→技术路线最优 |
| 📜「辞海无涯」 | 🤝人脉型、⚖️平衡型 | +10% | 文科背景→人脉/管理路线 |
| 🔧「匠心独运」 | ⚖️平衡型、💻技术型 | +10% | 实践能力→管理路线最优 |
| 🎨「灵感无界」 | 🚀冒险型、🤝人脉型 | +10% | 创造力→创业/创意路线 |
| 💪「不屈之心」 | 💪健康型、任意 | +10% | 体能意志→通用加成 |
| 🌟「通才印记」 | 任意 | +5% | 全能→最佳适配 |

**硬锁死影响**：
- 选物理方向的玩家：不点📐数理之道 → 计算机/医学等专业不可报 → 无法进入💻技术型职业（除非通过考研转专业，需额外2年+压力+20）
- 选历史方向的玩家：完全无法进入💻技术型职业（理工科专业全部不可报），但🤝人脉型和🚀冒险型完全开放
- 跨方向转行补偿：如果大学期间通过考研转换方向（如历史→法硕），可解锁对应职业路线，但工资系数×0.8（转行代价）

### 职场装备共鸣联动（不变）
- 天赋树T2/T3节点点亮 → 职场对应标签共鸣+1
- 与V6.1协同系统无缝衔接
- 职场第一年KPI额外+2分（天赋树有终极节点者）

## 三、工作系统（8大专业方向）

### 3.1 工作模式设计

**延续上学模式，但添加职业特色**：
- 属性成长 → 职业技能成长
- 考试 → KPI考核/项目评审
- 事件 → 职场事件
- 装备 → 职业工具/证书

```javascript
const WORK_SYSTEM = {
  // 工作阶段（22-30岁，8年）
  duration: 8, // 年
  
  // 工作模式
  mode: {
    // 每年4个季度（类似学期）
    quarters_per_year: 4,
    // 每个季度事件
    events_per_quarter: 3,
    // KPI考核（每年1次）
    kpi_per_year: 1,
  },
  
  // 薪资系统
  salary: {
    base: (player) => {
      // 基础薪资 = 专业系数 × 大学档次系数 × 城市系数
      const major_coeff = player.major_coeff; // 专业薪资系数
      const uni_coeff = player.university_tier_coeff; // 大学档次系数
      const city_coeff = player.city_coeff; // 城市系数
      return 5000 * major_coeff * uni_coeff * city_coeff;
    },
    // 年终奖（根据KPI）
    bonus: (player, kpi_score) => {
      return player.salary_base * (0.5 + kpi_score * 0.01);
    },
    // 涨薪（每年根据表现）
    raise: (player, performance) => {
      return player.salary_base * (0.05 + performance * 0.005);
    }
  },
  
  // KPI考核（类似考试）
  kpi: {
    formula: '基础分60 + 智力×0.2 + 社交×0.15 + 专业技能×0.3 + 装备加成 - 压力×0.1',
    // KPI等级
    grades: {
      'S': { min: 90, bonus: 0.3 }, // 年终奖+30%
      'A': { min: 80, bonus: 0.2 }, // 年终奖+20%
      'B': { min: 70, bonus: 0.1 }, // 年终奖+10%
      'C': { min: 60, bonus: 0 },   // 无奖金
      'D': { min: 0, bonus: -0.1 }, // 扣工资10%
    }
  }
};
```

// 录取概率计算
const admission_probability = {
  // 基础概率
  base: 0.5 + (player_score - uni_min_score) / 20,
  
  // 属性综合加成（无随机）
  attribute_bonus: {
    intelligence: (player.intelligence - 60) * 0.002,
    talent: (player.talent - 5) * 0.01,
    equipment: player.equipment_bonus * 0.01,
    family: (player.family_coeff - 1) * 0.05,
  },
  
  // 策略修正
  strategy_modifier: {
    RUSH: -0.15,    // 冲-15%
    STEADY: 0,      // 稳0%
    SAFE: +0.10,    // 保+10%
  },
  
  // 志愿顺序加成
  order_bonus: {
    1: 0,
    2: 0,
    3: 0.05,
    4: 0.10,
    5: 0.15,
    6: 0.20,
  },
  
  // 专业热度
  major_heat: {
    hot: -0.10,     // 热门-10%

    cold: +0.10,    // 冷门+10%
  },
  
  // 服从调剂
  adjustment: {
    true: +0.15,    // 服从+15%
    false: 0,
  },
  
  // 最终概率（限制在0.05-0.95）
  final: (base + attr_bonus + strategy + order + heat + adj) => {
