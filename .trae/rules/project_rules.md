# 重启我的高中人生 - 项目规范

## 一、每次改完代码必须做的事
1. 检查所有修改文件的编译错误（用 GetDiagnostics）
2. 确认没有 CS 错误，warning 也要尽量消除
3. 验证逻辑流程是否完整

## 二、UI 设计规范
1. **不要出现程序调试文字**：如"临时版先确认身份与输入流程"、"理性稳稳型"、"学习+2"等，这些都是开发占位符，正式版必须去掉
2. **难度信息只通过颜色展示**：省市难度用绿/黄/红三色区分，不要显示具体分数
3. **整屏界面用返回按钮，弹窗用关闭按钮**：不要同时存在两个，功能重复
4. **NEW 标签只在真正替换后才显示**：首次抽取/首次进入不显示 NEW

## 三、数据持久化
1. `HasSaveData` 是判断是否有存档的核心标志
2. 注销/重置时务必设置 `HasSaveData = false` 并调用 `PlayerPrefs.DeleteAll()`
3. `ResetState()` 不自动清除 `HasSaveData`（设计意图：保留新游戏开始时的状态）
4. 进入主界面后立即调用 `SaveGame()` 写入磁盘

## 四、素材路径规范
1. 学期插画：`UI/学期/{男|女}/{学期名}.png`（如 `UI/学期/男/高一上.png`）
2. 毕业照：`UI/学期/{男|女}/高中毕业.png`
3. 结局图：`UI/结局/{男|女}end_{结局类型}.png`（如 `UI/结局/男end_doctor.png`）
4. 男女形象：`UI/男.png`、`UI/女.png`
5. 所有图片必须为 Sprite 导入类型（textureType: 8, spriteMode: 1）

## 五、常见 Bug 预防
1. **`AddComponent` 只调用一次**：不要连续调用 `AddComponent<VerticalLayoutGroup>().property` 来设置属性，应该先存为变量再配置
2. **学期进度显示**：`Home` 阶段也要显示学期信息（如"高一上 · 即将开始"），不要留空
3. **按钮互斥**：登录界面有存档时只显示"返回校园"，无存档时只显示"重启我的高中人生"
4. **ScrollRect 的垂直滚动**：`verticalNormalizedPosition` 1=顶部，0=底部

## 六、学期流程
1. 进入学期界面 → 显示学期信息 → 点"结算本学期" → 成绩单 → 回到主界面
2. 每个学期完成后自动生成 ABC 成绩并保存到 `SemesterGrades`
3. 主界面可横滑查看已完成学期的成绩

## 七、难度分级（省市）
- 轻松（1-2分）：绿色 - 自主命题、专项计划
- 正常（3分）：蓝色 - 新高考Ⅱ卷
- 困难（4-5分）：红色 - 新高考Ⅰ卷（河北/山东/河南等5分）
